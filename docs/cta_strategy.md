## 設計思想

- **主戦場は直接営業。LPは対面営業の補助＋紹介経由の受け皿**
- UKEファイル診断はLP不使用、対面営業専用（信頼・法的ハードルが高すぎる）
- CTAは各記事に1つだけ。複数置くと迷って離脱する
- プロダクト開発中のため、CTA出口は「無料モニター登録」（ウェイトリスト）

## メインCTA：無料PDFレポート＋無料モニター登録

```
note記事（24本）
  │
  │ 記事末尾CTA
  ▼
Brevo LP（lp.barbara.co.jp）
  │
  ├─ メインCTA：無料PDFレポートダウンロード
  │   メールアドレス入力 → PDF自動送信
  │   → フォローメール（ステップメール）で無料モニターへ誘導
  │
  └─ サブCTA：無料モニター登録（BOFU記事からのみ直接誘導）

並行チャネル（主力）：
  ORCA管理機構・ベンダー紹介 → 対面でUKE診断デモ → 無料モニター → 有料契約
```

## 無料PDFレポート構成（6〜8ページ）

| ページ | 内容 | 狙い |
|--------|------|------|
| 1 | 表紙＋「3ヶ月来ていない患者、何人いますか？」 | フックで掴む |
| 2-3 | 患者離脱の実態（データ）＋機会損失の計算例 | 「うちもこうかも」と思わせる |
| 4-5 | 離脱を防いでいるクリニックがやっていること | 具体的な改善イメージ |
| 6 | ミルカルテの紹介（発見→実行→測定の一貫提供） | ここで初めてサービス名 |
| 7 | 無料モニター募集の案内 | CTA：モニター登録 |

## 無料モニターの位置づけ

- 対外的な表現：**「無料モニターにご協力いただけるクリニックを募集しています」**
- 実質＝Tier 0（無料体験レポート）。院長のUKEデータで実際のレポートを無料作成
- モニター登録リスト＝PMF検証の見込み客リスト
- 対面営業時にも「今、無料モニターを募集していまして」と使える

### モニター条件（確定）

- **月1回15分のフィードバック面談**を必須条件に含める
  - 目的①：院長のコミットを確保する
  - 目的②：コンテンツのネタ収集（リアルな声）
- コンテンツ → モニター → フィードバック → コンテンツのループを回す
