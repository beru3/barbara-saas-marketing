## 設計思想

- **主戦場は直接営業。LPは対面営業の補助＋紹介経由の受け皿**
- UKEファイル診断はLP不使用、対面営業専用（信頼・法的ハードルが高すぎる）
- CTAは各記事に1つだけ。複数置くと迷って離脱する
- プロダクト開発中のため、CTA出口は「初期参画院の募集」（共創枠）
- **顧客を「消費者」ではなく「共創者」として迎える** — IKEA効果の最大化

## 初期参画院の位置づけ

- 対外的な表現：**「サービスの設計に参加いただける第1期の先生を募集しています」**
- 実質＝Tier 0（無料体験レポート）＋共創パートナー。院長のUKEデータで実際のレポートを無料作成
- 初期参画院リスト＝PMF検証の共創パートナーリスト
- 対面営業時にも「今、第1期の先生方を募集していまして」と使える
- **「テストしてください」ではなく「一緒に作ってください」の姿勢**

### 参画条件（確定）

- **月1回15分のフィードバック面談**を必須条件に含める
  - 目的①：院長のコミットを確保する
  - 目的②：コンテンツのネタ収集（リアルな声）
  - 目的③：プロダクト設計への実質的な影響力を持たせる
- コンテンツ → 初期参画院 → フィードバック → コンテンツのループを回す

## 共創型マーケティングの3原則

### 原則1: 未完成を「特権」として提供する

- 使う語彙：「第1期の先生方」「立ち上げにご一緒いただく先生」「設計に参加する」
- 使わない語彙：「ベータ」「テスター」「モニター」「未完成品」
- 希少性設計：第1期は10院程度の少数限定（1院ずつ丁寧に向き合うため）

### 原則2: 初期参画院への特権（4軸）

| 軸 | 確定事項 | 検討中 |
|----|---------|--------|
| **時間** | 正式リリース前の先行利用 | 優先実装枠の具体的な本数・頻度 |
| **意思決定** | 月1回のフィードバックでロードマップに影響する | 投票権・命名権の具体ルール |
| **内部アクセス** | 全参画院の匿名ベンチマーク共有 | 成功打ち手DBの形式・更新頻度 |
| **経済** | 第1期価格の永久ロック | 紹介ディスカウントの具体率 |

### 原則3: 葛藤・失敗の実況中継

- 鉄則：「迷っています」ではなく「トレードオフを整理した上で経営判断の観点から意見を求める」姿勢
- 効くテーマ例：
  - 「離脱」の定義論争（何日来院なしを離脱とするか）
  - ROI試算の透明化（介入コスト vs 回収率）
  - 失敗開示（誤った仮説をデータが覆した過程）
  - チャネル選択（LINE / SMS / はがき / 電話の実数比較）
  - 価格モデルの公開議論

## 最初の10院（初期参画院）獲得チャネル（優先順）

| 優先度 | チャネル | 期待獲得数 |
|--------|---------|----------|
| 1 | ORCA管理機構経由の紹介 | 3〜5院 |
| 2 | ORCA導入ベンダー経由 | 2〜3院 |
| 3 | 知人・ネットワーク直接営業 | 2〜3院 |
| 4 | note記事経由 | 0〜1院 |

> 10院すべてが共創枠。「通常顧客」と「テスト顧客」を分けない。全員がサービス設計のパートナー。
