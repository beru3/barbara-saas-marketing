## ターゲット設定

### 見込み客像

| 項目 | 内容 |
|------|------|
| 診療科 | 内科（生活習慣病メイン）＋整形外科 |
| 規模 | 外来クリニック（中小）、ORCA導入約14,000施設 |
| 主な痛み | 患者が離脱していることに気づいていない |
| 特徴 | 経営・IT・マーケが苦手、データより勘で動く |

### 診療科別の攻め方

- **内科**: 施設数を稼ぐ（薄く広く）。慢性疾患の定期通院構造が明確。離患閾値: 45日超
- **整形外科**: 単価を稼ぐ（ROIで攻める）。リハビリ込みで内科の2〜4倍。離患閾値: 14日超

### ターゲットの深層心理

- 「忙しい」が口癖で経営指標を見ていない
- 新患獲得に意識が向き、既存患者のフォローが手薄
- 近隣クリニックとの比較を避ける傾向
- スタッフ依存・属人化に課題感がない

### ターゲットの情報行動（重要な前提）

院長は「能動的に検索しない」のではなく「検索する時間も習慣もない」。

院長が検索する瞬間：
- レセコン操作中に困った時
- 制度改定で慌てた時
- 補助金を探している時

→ この瞬間だけ検索行動が発生する

## プロダクト概要（マーケティングに必要な範囲）

### コアバリュー

> 競合（CLINIC BOARD等）が「分析提供」で止まるのに対し、**発見 → 実行（DM代行・LINE運用）→ 効果測定**まで一貫提供。

### 料金Tier

| Tier | 月額 | 主な機能 |
|------|------|---------|
| Tier 0 | 0円（初回のみ） | 無料体験レポート1回 → 「気づき」の体験設計 |
| Tier 1 | ¥14,800 | 月次レポートPDF＋総合スコア＋機会損失額＋改善推奨3項目 |
| Tier 2 | ¥39,800 | ＋離患リスク患者CSV＋アクションプラン詳細＋競合比較 |
| Tier 3 | ¥79,800 | ＋郵便DM代行（月50通）＋LINE運用＋実回収額レポート |
| Tier 4 | ¥149,800 | ＋週次モニタリング＋専任担当＋Web広告連動 |

### 価格の伝え方（5ステップ）― 全コンテンツ・営業資料で統一

1. 「先生の院の推定機会損失は月○○万円です」
2. 「このうち現実的に取り戻せるのは○○万円です」
3. 「サービス料はその回収額の○○%です」
4. 「月額にすると○○円です」

→ **「月10万円です」とは絶対に言わない**

### 競合との差別化

| 競合 | できること | バーバラの優位性 |
|------|----------|----------------|
| CLINIC BOARD | ベンチマーク提供 | 実行代行なし・API連携なし |
| レセコンアナライザー | データ分析 | LINE/DM代行なし |
| AI-Board | 分析ダッシュボード | WebORCA API未活用 |

### Tier 0営業フロー（コンテンツのトーンに影響する重要事項）

- Page1だけ先に見せる → 「先生の院のスコアが出ました」
- 説明しない。質問に答えるだけ
- 「来月もこの数字が見たければ月額14,800円で継続できます」→ それだけ言って帰る

→ **売り込み感ゼロがブランドの根幹**

## チャネル戦略（確定）

### 優先順位

| 優先度 | チャネル | タイミング | 目的 |
|--------|----------|------------|------|
| 1 | note（SEO資産） | 今すぐ | 信頼構築・長期流入 |
| 1 | **X（Twitter）** | 今すぐ | 認知・接触・即時反応、noteへの導線 |
| 2 | 固有名詞SEM | noteがインデックスされ始めたら | 検索意図が明確な層の刈り取り |
| 3 | Facebook広告 | 予算余裕が出てから検証 | 認知拡大（ROI要検証） |

### X × note ハイブリッド戦略（確定）

| チャネル | 役割 | 頻度 |
|----------|------|------|
| **X** | 認知・接触・即時反応、モニター告知の拡散 | 週3〜5投稿 |
| **note** | 信頼・成約、長文分析、CTA置き場 | 週1投稿 |

**導線：** Xで引っかける → noteで納得させる → LPで申込

**運用の鍵：**
- 反応数ではなく**投稿継続数をKPI**にする（行動は制御できる）
- Xで医師に売り込み臭を出さない（まず役立つ発信を半年、募集はその後）
- 最初の1院が決まったらその事例を記事化（10記事の抽象論より強い）

**時間軸の覚悟：** 反応が出始めるのは早くて3ヶ月、現実的には半年。半年まで覚悟済み。

### 固有名詞SEMのキーワード設計

| 検索タイミング | キーワード例 |
|---|---|
| レセコン操作中に困った | 「ORCA エラー」「ORCA レセプト 返戻」 |
| 制度改定で慌てた | 「2026年 診療報酬改定 内科」「オンライン資格確認 エラー」 |
| 補助金を探している | 「IT導入補助金 クリニック 2026」 |
| 比較検討中 | 「ORCA 電子カルテ 連携」「WebORCA 使い方」 |

特徴：CPC競合が少ないニッチワードで費用対効果が高い

### Facebook広告の留意点

- 40〜50代開業医はFacebookユーザー。職業ターゲティング「医師」が可能
- 罠：院長のFB利用は「同期との近況確認」目的。認知は取れるが商談化率が低い

## サイト構成

```
note記事（24本）
  │
  │ 記事末尾CTA
  ▼
Brevo LP（lp.barbara.co.jp）
  ├─ メインCTA：無料PDFレポートダウンロード → ステップメールで無料モニターへ誘導
  └─ サブCTA：無料モニター登録（BOFU記事からのみ直接誘導）

並行チャネル（主力）：
  ORCA管理機構・ベンダー紹介 → 対面でUKE診断デモ → 無料モニター → 有料契約
```

### note vs LP の役割分担

| 項目 | note | Brevo LP |
|------|------|----------|
| 役割 | 記事投稿（集客・信頼構築） | LP＋フォーム＋フォローメール自動配信 |
| 初期コスト | ほぼゼロ | 無料プランあり |
| SEO初速 | 高い（ドメインパワーを借用） | SEO不要（note記事から誘導） |
| リスト取り | 不向き | フォーム＋メール自動化を一括管理 |

Phase 1はnote一択。LP・CTA・フォローメールはBrevoで構築。

### noteの役割の再定義（2026年4月決定）

- note記事の主目的は「直接営業時の信頼の裏付け」であり、直接コンバージョンチャネルではない
- 最初の10施設はORCA管理機構・ベンダー紹介・直接営業で獲得する（note経由は0〜1施設想定）
