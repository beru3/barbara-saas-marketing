# コンテンツ制作ガイドライン

> マーケティング戦略の一部。全体構成は [CLAUDE.md](../CLAUDE.md) を参照。

---

## コンテンツの4つのエッセンス

全記事に最低1つ含める。

| エッセンス | 目的 |
|------------|------|
| マーケティング | 患者獲得・維持の視点を持たせる |
| IT | データ活用の重要性を理解させる |
| 経営 | 数字・KPIで考える習慣を作る |
| 組織マネジメント | 属人化リスクへの気づきを促す |

---

## 記事構成テンプレート

1. タイトル（痛みを刺す・数字を入れる・院長語彙で書く）
2. リード文（共感から入る・100〜150字）
3. 問題提起（院長の現状を描写）
4. 原因分析（データ・事例を交えて）
5. 解決策の方向性（ミルカルテへの自然な導線）
6. まとめ
7. CTA（無料PDFレポート or 無料モニター登録）

## 投稿ルール

| 項目 | 内容 |
|------|------|
| 頻度 | 週1本（月4本） |
| 投稿曜日 | 火曜日固定 |
| 期間目標 | 最低6ヶ月・24本 |
| 末尾CTA | 全記事にPDFレポート or モニター登録のCTAを設置 |

## フェーズ設計

| フェーズ | 期間 | 本数 | 目的 |
|----------|------|------|------|
| 基盤構築 | 1〜3ヶ月 | 月4本 | インデックス獲得 |
| 認知拡大 | 4〜6ヶ月 | 月4本 | 検索流入開始 |
| 刈り取り | 7ヶ月〜 | 月2〜4本 | リスト取り・問い合わせ |

## 制作フロー

古家が話す・方向性を決める → Claudeが構成・文章化 → 古家が確認・修正 → note投稿

目的：疲弊を最小化しながら継続的に発信する。

## ネタ切れ防止：4軸の掛け算

### 軸1：切り口の掛け算

同じテーマでも切り口を変えれば別記事になる。

| 切り口 | 例（テーマ：患者離脱） |
|--------|----------------------|
| 失敗談 | 「離脱を放置して年商が200万円減ったクリニックの話」 |
| 数字検証 | 「離脱患者10人で年間いくらの機会損失か計算してみた」（#9） |
| 他業界移植 | 「飲食店のリピーター施策をクリニックに移植する」 |
| 逆張り | 「新患獲得をやめたクリニックが繁盛した理由」 |
| FAQ化 | 「『うちは地域密着だから大丈夫』と言う院長に伝えたいこと」 |

### 軸2：季節・制度カレンダー

| 時期 | ネタ |
|------|------|
| 4月 | 新年度・診療報酬改定対応 |
| 6月 | 特定健診シーズン開始 |
| 9〜10月 | インフルエンザ予防接種シーズン |
| 12〜1月 | 年末年始の患者離脱ピーク |
| 随時 | IT導入補助金の公募時期 |

### 軸3：他業界翻訳シリーズ【優先度最高】

院長が自院の課題を「自分ごと」として認識しやすい切り口。

| 他業界 | クリニックへの翻訳 |
|--------|-------------------|
| 飲食業 | リピーター管理・客単価分析 → 再診率・診療単価 |
| 美容院 | 次回予約・DM施策 → 患者フォロー・リコール |
| フィットネス | 休眠会員の掘り起こし → 休眠患者の再来院促進 |
| SaaS業界 | チャーンレート（解約率） → 患者離脱率 |

### 軸4：読者QA・反論処理シリーズ

院長が持つ典型的な反論や疑問を記事化する。

| 反論・疑問 | 記事化の方向性 |
|-----------|---------------|
| 「うちは口コミで十分」 | 口コミだけに頼るリスクをデータで示す |
| 「ITは難しい」 | ORCAデータの活用は特別なスキル不要であることを伝える |
| 「スタッフに任せている」 | 属人化リスクを具体的なシナリオで見せる |
| 「患者は減っていない」 | 「減っていない」と「増えていない」の違いを可視化する |
| 「うちは地域密着だから大丈夫」 | 地域密着でも離脱は起きている事実をデータで示す |
